## 修复更新的功能
1. 将Action构建条件改为手动 
2. 修复支持代理，并且同时支持：
	- 自动使用系统代理
	- 手动代理(需要再终端中导入http_proxy|https_prox|all_proxy等代理参数)
3. 新增图片如果存在时，就不重新下载的功能。
4. 修复不能现在标记为受限或具有攻击性的图库。
5. 添加IP被限制的警告提示。
6. 添加参数：-r|--retry,允许一直重试失败的任务，直到所有的任务都完成。
	- 注意：重试允许条件-如不合适可以根据情况修改handler.rs文件中is_retryable()函数中的一些条件
7. 添加参数：proxy-mode设置代理方式，支持none|http|socks
	- none：使用默认配置，即不手动管理代理配置
	- http：同时添加Proxy.http和Proxy.https两种代理配置
	- socks：使用Proxy.all设置socks代理配置，并且Proxy.all支持所有的协议，如：http,https与其他协议的请求
8. 添加参数：proxy来手动指定代理服务地址。
	- 注意：只支持http, https, socks5, socks5h协议的代理服务地址
9. 添加参数：convert-socks5h会将socks5协议的地址转化成socks5h协议的地址。







## 大的更改：
reqwest 从 0.9.12 升级到 0.11，其中需更改的内容：
1. 把 Cargo.toml 改成：
```
//原来：
reqwest = "0.9.12"
url = "1.7.2"

//更改后：
reqwest = { version = "0.11", features = ["blocking", "socks", "rustls-tls"] }
url = "2.5.8"

//解释：
blocking：使用老旧的block方式处理网络请求
socks+rustls-tls：启用对Proxy::all("socks5://...")的支持


```
⚠️ 注意：0.11 默认是 async，你现在代码是同步版本，所以必须加：
```
features = ["blocking"]
```
2. 然后你需要：
```
use reqwest::blocking::{Client, Response};
```
并且所有：
```
reqwest::Response
```
改成：
```
reqwest::blocking::Response
```
升级后：
```
e.is_timeout()
e.is_connect()
e.status()
```
全部可用。
